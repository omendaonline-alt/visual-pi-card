1. Abstract & Vision

Visual Pi Card is a next-generation digital payment card platform built exclusively for the Pi Network ecosystem. It provides Pi holders with a visual, card-based interface to manage, spend, exchange, and grow their Pi (π) cryptocurrency across six distinct card tiers — each with unique pricing models, rewards, and capabilities.

The platform currently operates in Pi Testnet mode, allowing users to explore all card features using testnet Pi. All activation fees, transactions, rewards, and exchange operations are simulated on the Pi SDK sandbox. Once Pi Network transitions to open mainnet, Visual Pi Card will migrate to production with real Pi transactions.

Core Principle: Each card tier has its own independently stable or market-driven Pi price. The balance deposited on the main platform is shared, but the USD equivalent displayed depends on which card the user selects. This means 1,000 π shows different dollar values on each card — by design.

Key Features

  • 6 Card Tiers — Visa, Mastercard, Gold, Platinum, Black, Amex — each with unique logic
  • 24-Word Passphrase Security — Every card is locked behind wallet-grade authentication
  • Owner Identity Verification — Card holder name, Pi username, and wallet address displayed on every card
  • Pi SDK Integration — Testnet authentication and payment via Pi Browser
  • Progressive Web App (PWA) — Installable, offline-capable, mobile-first design

10. Security & Authentication

Layer 1: 24-Word Passphrase

Every card page is protected by a full-screen passphrase unlock overlay (z-index 9999). The user must enter exactly 24 words (whitespace-separated) to access any card. This mirrors Pi Wallet’s non-custodial security model. The unlock state is saved in sessionStorage so re-entry isn’t required during the same session.

Layer 2: Activation Gate

Cards with activation fees (Gold: 100 π, Platinum: 10 π, Black: 120 π) have a second overlay (z-index 9998) that appears after the passphrase is accepted. Users must pay the activation fee to proceed. Activation is permanent (stored in localStorage).

Layer 3: Owner Identity

Every card displays the registered owner’s identity: full name, Pi username (@handle), and wallet address. If no registered user is found in localStorage, the card redirects to the registration page (index.html).

Layer 4: Pi SDK Authentication

The main registration flow uses Pi Browser SDK v2.0 (Pi.authenticate()) to verify the user is a real Pi pioneer. The authenticated username and UID are auto-filled into the registration form.

Client-Side Security Model

Visual Pi Card uses an intentionally client-side architecture for the testnet phase. All balance, transaction, and activation data is stored in the browser’s localStorage. This design allows:

  • Zero backend database — no server-side user data to breach
  • Instant offline access (PWA)
  • Easy testnet experimentation without real funds at risk

For mainnet, a server-side database and Pi Payments API integration will replace localStorage.
